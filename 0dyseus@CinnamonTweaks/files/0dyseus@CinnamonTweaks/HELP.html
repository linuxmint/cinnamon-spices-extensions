Tilpasning af pop op-menuer

Panelmenuers opførsel

Bemærk: Denne indstilling påvirker kun opførslen af menuer tilhørende panelprogrammer placeret på ethvert panel.

  • Emulér Gnome Shells opførsel: Når en menu er åben i Gnome Shell, og musemarkøren flyttes til en anden knap i toppanelet, så vil menuen tilhørende en knap, som musemarkøren hviler på, automatisk åbne, uden at der klikkes på den. Med denne indstilling aktiveret kan denne opførsel reproduceres i Cinnamon.
  • Spis ikke klik: Når man i Cinnamon åbner et panelprograms menu og så klikker på et andet panelprogram for at åbne dets menu, vil det første klik som standard lukke det første panelprograms menu, og det næste klik åbne det andet panelprograms menu. Med denne indstilling aktiveret kan man med et enkelt klik åbne ethvert panelprograms menu, selvom et andet panelprograms menu er åbent.

Tilpasning af værktøjstip

  • Undgå at musemarkør overlapper værktøjstip: Værktøjstip i Cinnamons brugergrænseflade vises med øverste venstre hjørne ved musemarkørens position. Dette kan føre til, at værktøjstip overlapper med musemarkøren. Denne tilpasning viser værktøjstippet med nederste venstre hjørne ved musemarkøren (omtrentlig) og mindsker derfor sandsynligheden for overlap med musemarkøren.
  • Forsinkelse for visning af værktøjstip: Krystalklar.

Tilpasning af meddelelser

  • Aktivér animation ved åbning/lukning af meddelelser: Krystalklar.
  • Placering af meddelelser: Meddelelser kan vises øverst til højre (systemstandard) eller nederst til højre.
  • Afstand til panel
    • Meddelelser vist øverst til højre: Dette er afstanden mellem toppanelets (fra toppen af skærmen hvis der ikke er noget toppanel) nederste kant til den øverste kant i meddelelsesvinduet.
    • Meddelelser vist nederst til højre: Dette er afstanden mellem bundpanelets (fra bunden af skærmen hvis der ikke er noget bundpanel) øverste kant til den nederste kant i meddelelsesvinduet.
  • Meddelelsesvinduets højre margen: Som standard defineres meddelelsesvinduets højre margen af det aktuelle tema. Hvis denne indstilling sættes til en anden værdi end 0, kan højre margen tilpasses.

Tilpasning baseret på udvidelsen kaldet Cinnamon Maximus af Fatih Mete med nogle indstillinger fra udvidelsen til gnome-shell kaldet Maximus NG af Luis Pabon. Denne tilpasning tillader fjerne af titellinjer i maksimerede, halv-maksimerede og side om side udlagte vinduer.

Afhængigheder

Denne tilpasning kræver to kommandoer tilgængelige i systemet (xprop og xwininfo) for at virke.

  • Debian-baserede distributioner: Disse kommandoer er tilgængelige i pakken x11-utils. I Linux Mint er denne pakke allerede installeret.
  • Archlinux-baserede distributioner: Disse kommandoer er tilgængelige i pakkerne xorg-xprop og xorg-xwininfo.
  • Fedora-baserede distributioner: Disse kommandoer er tilgængelige i pakken xorg-x11-utils.

Advarsler

  • Vinduer udsmykket på klientsiden og WINE-programmer påvirkes ikke af denne tilpasning.
  • Luk alle vinduer som tilhører et program, som vil blive tilføjet til programlisten, og før denne tilpasnings indstillinger anvendes.
  • Som en forholdsregel for at undgå problemer så luk alle åbne vinduer, inden du aktiverer og konfigurerer denne tilpasning. Aktivér og konfigurér derefter denne tilpasning og log af og log på igen.

Kendte problemer

  • Usynlige vinduer Nogle gange kan vinduer, konfigurerede til at få fjernet deres titellinje, blive usynlige. Programmets ikon kan stadig ses i panelet, men når der klikkes på det for at fokusere vinduet, vil det usynlige vindue blokere klikkene, som om det var synligt. For at løse dette skal vinduet bringes ud af den maksimerede tilstand (det vil blive synligt igen) og lukkes. Når det åbnes igen, skulle vinduet opføre sig normalt.
  • Programmer uden titellinje: Nogle gange kan et program være uden titellinje, efter det er bragt ud af maksimeret tilstand. Genstart af programmet vil tillade, at titellinjen kan fjernes og tilføjes.

Alternativ

Der er en alternativ måde at skjule alle maksimerede vinduers titellinje på: ved at redigere dit Metacity-tema. Det virker uendeligt meget bedre og uden nogen af de problemer, som denne tilpasning i denne udvidelse har.

  • Gå til mappen /Sti/Til/Dit/Tema/metacity-1 og redigér filen metacity-theme-3.xml. Hvis denne fil ikke findes i dit tema, så er der en kaldet metacity-theme-2.xml eller metacity-theme-1.xml. Vælg den med et højeste nummer.
  • Find elementet max (eller maximized eller normal_max eller normal_maximized) i frame_geometry. Det præcise navn afhænger af temaet.
  • Attributen has_title skal tilføjes og sættes til false (falsk), og alle egenskabernes værdier skal sættes til 0. Nogle temaer kan kræve tilføjelse af flere egenskaber, hvis værdier også sættes til 0, for at slippe helt af med titellinjen.
  • Her følger nogle eksempler på, hvordan Metacity-temaerne i Mint-X- og Mint-Y-temaerne skal redigeres.

Generelle problemer med udvidelsen

Programmer optræder ikke i vinduet Programvælger

Vinduet Programvælger viser kun programmer, som har en tilgængelig .desktop-fil, fordi disse programmerer er de eneste, som tilpasningerne, der kræver et program-ID (Flyt vinduer automatisk og Fjernelse af vinduers titellinjer), vil genkende og håndtere.

Ifølge Desktop Entry Specification kan man oprette en .desktop-fil til ethvert program, som ikke optræder i programlisten.

Udvidelsens indstillingsvindue

Fra udvidelsens indstillingsvindue kan alle indstillinger importeres, eksporteres eller nulstilles til standardværdierne.

  • For at kunne udføre disse handlinger skal indstillingsskemaet installeres i systemet. Dette sker automatisk, hvis udvidelsen installeres gennem Cinnamons modul til håndtering af udvidelser. Blev udvidelsen installeret manuelt, skal indstillingsskemaet også installeres manuelt. Dette gøres ved at gå til udvidelsens mappe og køre følgende kommando:
    • Kommando til at installere indstillingsskemaet: ./settings.py install-schema
    • Kommando til at afinstallere indstillingsskemaet: ./settings.py remove-schema
  • For at importere/eksportere indstillinger skal kommandoen dconf findes i systemet.

Oversættelse af panelprogrammer, skrivebordsprogrammer og udvidelser

  • Hvis denne udvidelse blev installeret gennem Cinnamons indstillinger, blev alle oversættelser automatisk installeret.
  • Hvis denne udvidelse blev installeret manuelt og ikke gennem Cinnamons indstillinger, kan alle oversættelser installeres ved at køre skriptet localizations.sh i en terminal åbnet i udvidelsens mappe.
  • Hvis der ikke er nogen oversættelse til denne udvidelse tilgængelig på dit sprog, kan du oprette en ved at følge de efterfølgende instruktioner. Wiki

弹出菜单调整

面板菜单行为

注意： 此设置仅影响属于位于任何面板上的小程序的菜单的行为。

  • 模拟Gnome Shell行为： 当在Genome Shell上打开菜单，然后将鼠标光标移动到顶部面板上的另一个按钮时，所悬停的按钮的菜单将会自动打开而无需点击。启用此选项后，可以在Cinnamon上重现相同的行为。
  • 不要“吃掉”点击： 默认情况下，当在Cinnamon上打开一个小程序的菜单，然后点击另一个小程序要打开其菜单时，第一次点击用于关闭第一次打开的菜单，然后必须再执行一次点击才能打开第二个小程序的菜单。启用此选项后，即使打开了另一个小程序的菜单，也可以直接打开任何小程序的菜单。

提示调整

  • 避免鼠标指针与提示重叠： Cinnamon用户界面上的提示与鼠标指针的左上角对齐。这导致提示与鼠标指针重叠。此调整将提示对齐到鼠标指针的右下角（大约），从而减少鼠标指针与提示重叠的可能性。此调整仅适用于低于3.2的Cinnamon版本。Cinnamon3.2.x已经可以更改工具提示的位置。
  • 提示显示延迟： 浅显易懂。

通知调整

  • 启用通知开/关动画： 浅显易懂。
  • 通知位置： 通知可以在屏幕的右上方（系统默认）或右下方显示。
  • 面板距离：
    • 对于屏幕右上方显示的通知： 这是指顶部面板的底部边框（如果没有顶部面板，则从屏幕顶部）到通知弹窗的顶部边框之间的距离。
    • 对于屏幕右下角显示的通知： 这是指底部面板的顶部边框（如果没有底部面板，则从屏幕底部）到通知弹窗的底部边框之间的距离。
  • 通知弹窗右侧边距： 默认情况下，通知弹窗的右侧边距是由当前使用的主题所定义的。此选项，设为任何其他大于0（零）的值，允许设置自定义右侧边距，忽略由主题所定义的右侧边距。

窗口装饰移除

调整基于Fatih Mete的Cinnamon Maximus扩展和Luis Pabon的Maximus NGgnome-shell扩展中的一些功能。此调整允许从最大化/半最大化/平铺的窗口中移除窗口装饰。

注意： 如果您要选择的应用程序没有显示在应用程序选择器对话框中，请阅读此帮助文件中名为“应用程序未显示在应用程序选择器对话框**”的部分。

依赖关系

此调整需要(xprop和xwininfo)两个命令在系统上可用才能工作。

  • Debian系发行版： 这些命令由x11-utils软件包提供。Linux Mint已经安装了该软件包。
  • Archlinux系发行版： 这些命令由xorg-xprop和xorg-xwininfo软件包提供。
  • Fedora系发行版： 这些命令由xorg-x11-utils软件包提供。

警告

  • 客户端装饰的窗口和WINE应用程序不受此调整的影响。
  • 关闭属于要添加到应用程序列表的应用程序的所有窗口，然后应用此调整的设置。
  • 作为避免问题的通用规则，在启用和配置此调整之前，关闭当前打开的所有窗口，启用并配置此调整，然后注销并重新登录。

已知问题

  • 窗口不可见： 有时，配置为移除其装饰的应用程序窗口可能会变得不可见。应用程序的图标仍然可以在面板（任务栏）中看到，当单击以聚焦到其各自的窗口时，不可见的窗口将阻止点击，就像它是可见的。为了解决这个问题，窗口需要取消最大化（它将会再次变得可见）然后关闭。重新打开时，窗口应该会正常运行。
  • 应用程序卡住时取消装饰： 有些时候，一个应用程序即使在取消最大化之后也会卡住。重新启动应用程序将会恢复装饰和取消装饰其自身的能力。

Dependencias

Este ajuste requiere dos comandos disponibles en el sistema (xprop y xwininfo) para que funcione.

Problemas conocidos

  • Ventanas invisibles: A veces, las ventanas de aplicaciones que están configuradas para que sus decoraciones sean removidas se pueden volver invisibles. Para arreglar esto, la ventana necesita ser des-maximizada (se volverá visible) y luego cerrada. Cuando se reabre la aplicación, su ventana debería comportarse normalmente.
  • Aplicaciones atascadas sin decoraciones: A veces, una aplicación queda atascada sin decoraciones incluso después de volver a maximizarla. Reiniciado la aplicación hará que su capacidad para cambiar su estado decorada y sin decorar sea recuperada.

Beroenden

För att fungera kräver den här justeringen att två kommandon finns tillgängliga i systemet (xprop och xwininfo)

Varningar

  • Programfönster dekorerade av klientsidan och WINE-program, påverkas inte av denna justering.
  • Stäng alla fönster som tillhör de program som skall läggas till i programlistan, innan du tillämpar inställningarna i den här justeringen.
  • Som generell regel för att undvika problem, bör du stänga alla öppna fönster innan du aktiverar och konfigurerar denna justering och sedan logga ut/in när du är klar.

Kända problem

  • Osynliga fönster: Ibland kan programfönster som konfigurerats att ta bort dekorationen, bli osynliga. Programikonen kan fortfarande ses i panelen (aktivitetsfältet) och när den klickas för att fokusera respektive fönster, blockerar det osynliga fönstret klicken som om det var synligt. För att fixa detta, måste fönstret avmaximeras (det blir då synligt igen) och sedan stängas. När det öppnas igen, bör det uppföra sig normalt.
  • Programfönster som fastnat utan dekoration: Ibland kan ett programfönster fastna i odekorerat läge efter återgång från maximering. Omstart av programmet återställer förmågan att ta bort och återställa dekorationen.

Extension options details

Some tweaks have warnings, dependencies, limitations and or known issues that must be read and understood before a tweak is enabled. No worries, nothing *fatal* could ever happen.

Applets/Desklets tweaks

  • Ask for confirmation on applet/desklet removal: Instead of directly remove the applet/desklet from the context menus, it will ask for confirmation. This option doesn't affect the removal of applets/desklets from the Applets/Desklets manager in Cinnamon settings (there will be no confirmation).
  • Display "Open applet/desklet folder" on context menu for applets/desklets & Display "Edit applet/desklet main file" on context menu for applets/desklet: These options will add new menu items to the applets/desklets context menus. The place where this items will be located is chosen by the option Where to place the menu item?.

Hot Corners tweaks

This tweak is only available for Cinnamon versions lower than 3.2. Cinnamon 3.2.x already has hot corners delay activation.

  • Top left hot corner activation delay: Crystal clear.
  • Top right hot corner activation delay: Crystal clear.
  • Bottom left hot corner activation delay: Crystal clear.
  • Bottom right hot corner activation delay: Crystal clear.

Desktop area tweaks

  • Enable applications drop to the Desktop: This tweak enables the ability to drag and drop applications from the menu applet and from the panel launchers applet into the desktop.

Popup menus tweaks

Panel menus behavior

Note: This setting affects only the behavior of menus that belongs to applets placed on any panel.

  • Emulate Gnome Shell behavior: When a menu is open on Genome Shell, and then the mouse cursor is moved to another button on the top panel, the menu of the hovered buttons will automatically open without the need to click on them. With this option enabled, that same behavior can be reproduced on Cinnamon.
  • Don't eat clicks: By default, when one opens an applet's menu on Cinnamon and then click on another applet to open its menu, the first click is used to close the first opened menu, and then another click has to be performed to open the menu of the second applet. With this option enabled, one can directly open the menu of any applet even if another applet has its menu open.

Tooltips tweaks

  • Avoid mouse pointer overlapping tooltips: Tooltips on Cinnamon's UI are aligned to the top-left corner of the mouse pointer. This leads to having tooltips overlapped by the mouse pointer. This tweak aligns the tooltip to the bottom-right corner of the mouse pointer (approximately), reducing the possibility of the mouse pointer to overlap the tooltip. This tweak is only available for Cinnamon versions lower than 3.2. Cinnamon 3.2.x already has the position of the tooltips changed.
  • Tooltips show delay: Crystal clear.

Notifications tweaks

  • Enable notifications open/close animation: Crystal clear.
  • Notifications position: Notifications can be displayed at the top-right of screen (system default) or at the bottom-right of screen.
  • Distance from panel:
    • For notifications displayed at the top-right of screen: This is the distance between the bottom border of the top panel (if no top panel, from the top of the screen) to the top border of the notification popup.
    • For notifications displayed at the bottom-right of screen: This is the distance between the top border of the bottom panel (if no bottom panel, from the bottom of the screen) to the bottom border of the notification popup.
  • Notification popup right margin: By default, the right margin of the notification popup is defined by the currently used theme. This option, set to any value other than 0 (zero), allows to set a custom right margin, ignoring the defined by the theme.

Window focus tweaks

Tweak based on the gnome-shell extension called Steal My Focus by Valentin Dimitrov and another gnome-shell extension called Window Demands Attention Shortcut by awamper.

Some windows that demands attention will not gain focus regardless of the settings combination on Cinnamon settings. This option will allow you to correct that.

  • The activation of windows demanding attention...:
    • ...is handled by the system: Crystal clear.
    • ...is immediate: will force windows demanding attention to be focused immediately.
    • ...is performed with a keyboard shortcut: will focus windows demanding attention with a keyboard shortcut.
  • Keyboard shortcut: Set a keyboard shortcut for the option ...is performed with a keyboard shortcut.

Window Shadows tweaks

Tweak based on a Cinnamon extension called Custom Shadows created by mikhail-ekzi. It allows to modify the shadows used by Cinnamon's window manager (Muffin).

Note: Client side decorated windows aren't affected by this tweak.

Shadow presets
  • Custom shadows
  • Default shadows
  • No shadows
  • Windows 10 shadows

Auto move windows

Tweak based on the gnome-shell extension called Auto Move Windows by Florian Muellner. It enables the ability to set rules to open determined applications on specific workspaces.

Note: If the application that you want to select doesn't show up on the application chooser dialog, read the section on this help file called Applications not showing up on the applications chooser dialogs.

Windows decorations removal

Tweak based on the extension called Cinnamon Maximus by Fatih Mete with some options from the gnome-shell extension called Maximus NG by Luis Pabon. This tweak allows to remove the windows decorations from maximized/half-maximized/tiled windows.

Note: If the application that you want to select doesn't show up on the application chooser dialog, read the section on this help file called Applications not showing up on the applications chooser dialogs.

Dependencies

This tweak requires two commands available on the system (xprop and xwininfo) for it to work.

  • Debian based distributions: These commands are provided by the x11-utils package. Linux Mint already has this package installed.
  • Archlinux based distributions: These commands are provided by the xorg-xprop and xorg-xwininfo packages.
  • Fedora based distributions: These commands are provided by the xorg-x11-utils package.

Warnings

  • Client side decorated windows and WINE applications aren't affected by this tweak.
  • Close all windows that belongs to an application that is going to be added to the applications list and before applying the settings of this tweak.
  • As a general rule to avoid issues, before enabling and configuring this tweak, close all windows currently opened, enable and configure this tweak and then log out and log back in.

Known issues

  • Invisible windows: Sometimes, windows of applications that are configured to remove their decorations can become invisible. The application's icon can still be seen in the panel (taskbar) and when clicked to focus its respective window, the invisible window will block the clicks as if it were visible. To fix this, the window needs to be unmaximized (it will become visible again) and then closed. When reopened, the window should behave normally.
  • Applications stuck undecorated: Some times, an application will get stuck undecorated even after unmaximizing it. Restarting the application will recover its ability to decorate and undecorate itself.

General extension issues

Applications not showing up on the applications chooser dialogs

The application chooser dialog used by the settings window of this extension lists only those applications that have available .desktop files. Simply because these applications are the only ones that any of the tweaks that require an application ID (Auto move windows and Windows decorations removal) will recognize and handle.

Following the Desktop Entry Specification, one can create a .desktop file for any application that doesn't appear in the applications list.

Extension's settings window

From this extension settings window, all options can be imported, exported and/or reseted to their defaults.

  • To be able to perform any of these actions, the settings schema needs to be installed in the system. This is done automatically when the extension is installed from the Cinnamon extensions manager. But if the extension was installed manually, the settings schema also needs to be installed manually. This is achieved by simply going to the extension folder and launch the following command:
    • Command to install the settings schema: ./settings.py install-schema
    • Command to uninstall the settings schema: ./settings.py remove-schema
  • To import/export settings, the dconf command needs to be available on the system.
